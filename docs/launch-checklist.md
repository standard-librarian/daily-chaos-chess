# Launch Checklist

## Before public org launch

- Choose one flagship repo.
- Make the README understandable in under 30 seconds.
- Record a 30-60 second demo with no dead time.
- Verify a new user can hit first success in under 5 minutes.
- Collect feedback from 5-10 target users.
- Prepare a short launch post for X and a longer README-first launch post for
  GitHub.

## Public org contents

- flagship repo only
- `.github` community defaults
- optional landing page only if it can be completed in under one day

## First 30 days

- Measure stars, clones, setup attempts, and inbound issues.
- Track where users get stuck in the quickstart.
- Resist adding adjacent features until the main demo path is crisp.
