# Private Bake-Off Scorecard

Score each project from 1-5 after every meaningful iteration.

## Criteria

| Criteria | Traceproof | Replaygate | Notes |
| --- | --- | --- | --- |
| 30-second comprehension |  |  | Can a new visitor explain it back immediately? |
| 5-minute first success |  |  | Is setup fast enough for curious GitHub visitors? |
| Demo shareability |  |  | Would a 45-second clip earn reposts/stars? |
| Setup friction |  |  | Lower friction should score higher. |
| Pain intensity |  |  | Does the target user feel this problem sharply? |
| README clarity |  |  | Can the repo page carry attention on its own? |
| External feedback |  |  | How strong were reactions from real target users? |

## Decision rules

- Pick one flagship only.
- Bias toward the project with the better demo and clearer first-run success,
  not the broader roadmap.
- Do not launch the org publicly until one project clearly wins on both
  comprehension and setup speed.
